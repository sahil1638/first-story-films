# API Reference

This reference documents the Route Handlers under `src/app/api/**` and `src/app/auth/**`.

The application uses the Next.js App Router `route.ts` convention. Dynamic route parameters are promise-based in this Next.js version, so handlers resolve `params` with `await`.

## Conventions

| Concern | Standard |
|---------|----------|
| Auth | Session cookies issued by Supabase SSR |
| Authorization | Route handlers call role guards before business logic |
| Validation | Zod schemas in `src/lib/security/schemas.ts` |
| Errors | Most API routes use `handleApiError`; auth routes return explicit auth JSON |
| Rate limits | Database token bucket via `check_rate_limit`, called server-side with the service-role key |
| CSV export cap | 1,000 rows |
| PDF cache | Cache key includes document id and `updated_at` |

## Status Codes

| Status | Meaning |
|--------|---------|
| 200 | Request succeeded |
| 400 | Invalid request body, query, or route parameter |
| 401 | No valid session, invalid login, or missing credentials |
| 403 | Authenticated user lacks the required role |
| 404 | Resource not found |
| 405 | Unsupported HTTP method, generated by Next.js when not exported |
| 429 | Rate limit exceeded |
| 500 | Unexpected server error |
| 503 | PDF renderer unavailable or saturated |

## Auth

### `POST /api/auth/login`

Authenticates a user and creates a Supabase session.

Access: public.

Rate limit: `login:<ip>:<email>`, max 5 tokens, refill rate 5 attempts per 15 minutes.

Request:

```json
{
  "email": "user@example.com",
  "password": "Password123!"
}
```

Response:

```json
{
  "success": true
}
```

### `POST /api/auth/logout`

Signs out the current session.

Access: authenticated users.

Response:

```json
{
  "success": true
}
```

### `GET /auth/callback`

Handles Supabase auth callback codes and redirects to a sanitized local path.

Access: public.

Query parameters:

| Name | Required | Notes |
|------|----------|-------|
| `code` | Yes | Supabase auth code |
| `next` | No | Local redirect target; unsafe destinations fall back to `/dashboard` |

## Accounting Accounts

All accounting routes require `admin` or `manager`.

### `GET /api/accounting/accounts`

Returns paginated accounts.

Query parameters:

| Name | Values |
|------|--------|
| `page` | numeric string |
| `limit` | numeric string |
| `status` | `active`, `inactive`, `all` |
| `search` | text |
| `sortBy` | route-supported field |
| `sortOrder` | `asc`, `desc` |

### `POST /api/accounting/accounts`

Creates an account.

Request:

```json
{
  "name": "Cash",
  "openingBalance": 1000,
  "status": "active"
}
```

### `GET /api/accounting/accounts/[id]`

Returns one account by UUID.

### `PUT /api/accounting/accounts/[id]`

Updates account fields.

Request:

```json
{
  "name": "Cash on Hand",
  "openingBalance": 1000,
  "status": "active"
}
```

### `DELETE /api/accounting/accounts/[id]`

Deletes an account when business rules allow it.

### `GET /api/accounting/accounts/export`

Exports accounts as CSV.

Rate limit: `export:<user-id>`, max 5 tokens, refill 0.1 tokens per second.

## Accounting Entries

All entry routes require `admin` or `manager`.

Returns paginated ledger entries.

Query parameters:

| Name | Values |
|------|--------|
| `page` | numeric string |

## Masters

Master routes require `admin` or `manager`.

### `POST /api/masters`

Creates or updates a master record.

Supported `table` values:

| Table | Required fields on create |
|-------|---------------------------|
| `services` | `data.name` |
| `events` | `data.name` |
| `deliverables` | `data.title` |
| `agencies` | `data.company_name`, `data.person_name`, `data.contact_number` |
| `crew_members` | `data.name`, `data.contact_number` |

Example:

```json
{
  "table": "services",
  "id": "00000000-0000-0000-0000-000000000000",
  "data": {
    "name": "Cinematography",
    "description": "4K wedding film coverage",
    "status": "active"
  }
}
```

Agency and crew payloads may include `serviceIds`.

### `DELETE /api/masters`

Deletes a master record.

Query parameters:

| Name | Required | Values |
|------|----------|--------|
| `table` | Yes | `services`, `events`, `deliverables`, `agencies`, `crew_members` |
| `id` | Yes | UUID |

## PDFs

PDF routes require `admin`, `manager`, or `sales`.

Rate limits:

| Limit | Key | Policy |
|-------|-----|--------|
| Per-user PDF download | `pdf:<user-id>` | max 5 tokens, refill 0.2 tokens per second |
| Cache-miss render route | `pdf-route:<kind>` | max 10 tokens, refill 0.5 tokens per second |

### `GET /api/quotations/[id]/pdf`

Generates or returns the cached quotation PDF.

Response: `application/pdf`.

### `GET /api/orders/[id]/pdf`

Generates or returns the cached order agreement PDF.

Response: `application/pdf`.

### `GET /api/orders/[id]/payments/[paymentId]/receipt/pdf`

Generates or returns the cached payment receipt PDF.

Response: `application/pdf`.

## Maintenance

### `GET /api/maintenance/reconcile`

Reconciles role mismatches between `public.profiles.role` and Supabase Auth app metadata.

Access:

| Caller | Requirement |
|--------|-------------|
| Admin user | Valid admin session |
| Scheduler/service | `Authorization: Bearer <SUPABASE_SERVICE_ROLE_KEY>` |

Response:

```json
{
  "success": true,
  "message": "Reconciliation completed. Reconciled 0 drifting user roles.",
  "reconciled_count": 0
}
```
